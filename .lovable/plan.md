## Goal
Add an admin-only dashboard at `/admin/payment-links` that shows every Stripe price referenced by the app, where it's used, how it maps to a product/mode, and live status (active/inactive/upcoming expiration) pulled from Stripe.

## Data sources
1. **Static catalog** — `reports/payment-links.json` (already generated by `scripts/check-payment-links.ts`). Contains every referenced priceId, resolved mode/label, file, line, and status.
2. **Live Stripe data** — fetched via a new edge function `list-payment-links-status` which:
   - Reads the JSON report shipped with the build (or receives the price list from the client).
   - Calls `stripe.prices.retrieve(id, { expand: ['product'] })` for each unique price.
   - Returns per-price: `active`, `product.active`, `product.name`, `unit_amount`, `currency`, `recurring.interval`, `created`, and — for prices with a `metadata.expires_at` or coupons — an `expiresAt` field.
   - Admin-gated: verifies caller's JWT and `has_role(uid, 'admin')`.

Coupons/promotions aren't tied to a specific price, so we treat "upcoming expirations" as: (a) prices/products marked inactive in Stripe, and (b) any `metadata.expires_at` field on the price or product within the next 30 days. Anything beyond that is out of scope for a static price catalog.

## UI (`src/pages/admin/AdminPaymentLinks.tsx`)
Single table with columns:
- Price ID (copyable) + label from catalog
- Product name (Stripe) / mode (subscription/payment)
- Amount + interval
- Stripe status: Active / Inactive / Archived product
- Catalog status: ok / missing_in_map / bypasses_intake (from the JSON report)
- Referenced in (file:line, collapsed)
- Expires (if `metadata.expires_at` within 30 days — highlighted amber; past → red)

Above the table:
- Summary cards: Total prices, Active, Inactive, Expiring ≤30 days, Catalog failures
- Filter chips: All / Active / Inactive / Expiring / Failures
- Search box (priceId, product name, file)
- "Refresh from Stripe" button (re-invokes edge function; results cached client-side for the session)
- "Download report" button (fetches `/reports/payment-links.md` when available, else generates CSV client-side)

No writes from this page — read-only review.

## Files
- `supabase/functions/list-payment-links-status/index.ts` — new edge function, JWT-verified, admin-only, batched Stripe reads with basic in-memory memoization per invocation.
- `src/pages/admin/AdminPaymentLinks.tsx` — new page.
- `src/App.tsx` — register `/admin/payment-links` under the existing admin-guarded route group.
- `src/components/AdminNav.tsx` — add "Payment Links" nav entry.
- `scripts/check-payment-links.ts` — copy `reports/payment-links.json` into `public/reports/` during CI/build so the client can fetch the static catalog at runtime (small file, safe to ship: contains only priceIds and filenames, no secrets).

## Out of scope
- No editing prices/products from the dashboard.
- No historical time-series; current snapshot only.
- No Stripe coupon/promo tracking (they aren't 1:1 with prices).
